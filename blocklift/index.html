BlockLift Privacy Policy

Effective date: 2026-07-11

Summary

BlockLift is a local browser utility for visually selecting a page element and copying its sanitized HTML and matching CSS or exporting it as a local HTML file. It does not send personal data, page content, selected markup, styles or exports to a developer-operated server.

What data and content is processed

The extension processes the user-selected element's markup, browser-visible styles and accessible frame content on supported webpages. It also handles the local copy or export action explicitly chosen by the user.

How the data is used

This information is used only to display the selection overlay, sanitize the selected markup, prepare matching CSS, write a user-requested clipboard copy or generate a user-requested standalone local HTML export.

Sharing and transmission

The extension does not transmit selected markup, page styles, clipboard content or exports to the developer. Processing occurs locally in the browser. It does not make a developer-server request as part of copying or exporting an element; the webpage itself may continue its normal network activity.

Chrome storage and sync

The active or paused select-mode preference is stored through Chrome sync storage. Depending on the user's Chrome sign-in and sync settings, Chrome may synchronize this preference between the user's devices. It is not sent to the extension developer. The preference remains until the user changes it, clears extension data or uninstalls the extension.

Data sale and prohibited uses

The extension does not sell user data, use it for advertising, or use it for creditworthiness or lending purposes.

Remote code

The extension does not load or execute remotely hosted code. All executable JavaScript is included in the installed extension package.

Limitations

BlockLift does not bypass restricted browser pages or other access controls. Exports are best-effort for dynamic, Shadow DOM, canvas and script-driven components. Users are responsible for copying or exporting only content they own, manage or are permitted to use.

Retention and deletion

The developer does not retain extension data because no data is sent to a developer-operated server. Users can remove the stored preference by clearing the extension's data or uninstalling it. Clipboard content and exported files remain under the user's control and can be replaced or deleted locally.

Contact

For support or privacy questions, use the support URL on the extension's Chrome Web Store item page.
