InstaFetch Privacy Policy

Effective date: 2026-07-11

Summary

InstaFetch is a local browser utility that adds download controls for Instagram media already visible and accessible in the user's active browser session. It does not send personal data, browsing activity, media URLs or downloaded files to a developer-operated server.

What data and content is processed

The extension processes Instagram page content, media metadata and media or CDN URLs exposed to the active browser session. It may observe recent Instagram media requests already made by the browser to identify fallback URLs. It does not block, redirect, modify or upload those requests.

How the data is used

This information is used only to show download controls, identify media the user can already access, and save user-requested files or locally generated ZIP archives.

Sharing and transmission

The extension does not transmit processed page content, media URLs, download actions or downloaded files to the developer. When the user requests a download, the browser contacts Instagram's own website or media delivery domains directly. Local ZIP creation does not send media to a developer server.

Chrome storage and sync

The enabled or disabled overlay preference is stored through Chrome sync storage. Depending on the user's Chrome sign-in and sync settings, Chrome may synchronize this preference between the user's devices. It is not sent to the extension developer. The preference remains until the user changes it, clears extension data or uninstalls the extension. For fallback media detection, the extension also keeps up to 80 recently observed, tab-specific Instagram media or CDN URLs in chrome.storage.session. These entries are scoped to the browser session, used only while they are no more than three minutes old, and cleaned up during subsequent cache activity. They are not sent to the extension developer.

Data sale and prohibited uses

The extension does not sell user data, use it for advertising, or use it for creditworthiness or lending purposes.

Remote code

The extension does not load or execute remotely hosted code. All executable JavaScript is included in the installed extension package.

Limitations

InstaFetch does not bypass logins, private accounts, DRM, platform protections or rate limits. Users are responsible for downloading only content they own or are permitted to use.

Retention and deletion

The developer does not retain extension data because no data is sent to a developer-operated server. Recently observed media URLs may remain in session storage after their three-minute freshness window until later cache activity removes them or the browser session ends; they are not used after the freshness window. Users can remove stored extension data by clearing the extension's data or uninstalling it. Downloaded files remain in the location chosen by the user and can be deleted there.

Contact

For support or privacy questions, use the support URL on the extension's Chrome Web Store item page.
