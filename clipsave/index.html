ClipSave Privacy Policy

Effective date: 2026-07-11

Summary

ClipSave is a local browser utility for finding video source URLs already accessible to the browser on webpages the user can access. It does not send personal data, browsing activity, scan results, source URLs or downloaded files to a developer-operated server.

What data and content is processed

The extension processes video and source elements, embeds, media links, media metadata, accessible frame content and page-state source URLs exposed to the active browser tab.

How the data is used

This information is used only to list accessible video sources, apply the user's source filters, manage a temporary bulk-download queue and save a selected video file or accessible stream manifest URL when the user requests it.

Sharing and transmission

The extension does not transmit scan results, source URLs, preferences or downloaded files to the developer. When the user requests a download, the browser contacts the source website or media host directly.

Chrome storage and sync

Page-control and source-filter preferences are stored through Chrome sync storage. Depending on the user's Chrome sign-in and sync settings, Chrome may synchronize them between the user's devices. The complete bulk-download queue, including its items and source URLs, is kept in chrome.storage.local. These values are not sent to the extension developer. Preferences remain until changed, cleared or the extension is uninstalled. Completed queue data remains locally stored until a new queue replaces it, the user clears extension data, or the extension is uninstalled or updated in a way that removes extension data.

Data sale and prohibited uses

The extension does not sell user data, use it for advertising, or use it for creditworthiness or lending purposes.

Remote code

The extension does not load or execute remotely hosted code. All executable JavaScript is included in the installed extension package.

Limitations

ClipSave does not decrypt streams, merge segmented video, or bypass logins, DRM, private areas or platform restrictions. Users are responsible for downloading only media they own or are permitted to use.

Retention and deletion

The developer does not retain extension data because no data is sent to a developer-operated server. Completed queue items and their source URLs remain in local extension storage until a new queue replaces them, extension data is cleared, or the extension is uninstalled or updated in a way that removes extension data. Downloaded files remain in the location chosen by the user and can be deleted there.

Contact

For support or privacy questions, use the support URL on the extension's Chrome Web Store item page.
