PixelPull Privacy Policy

Effective date: 2026-07-11

Summary

PixelPull is a local browser utility for finding and downloading images already visible or exposed on webpages the user can access. It does not send personal data, browsing activity, page scans, image URLs or downloaded files to a developer-operated server.

What data and content is processed

The extension processes image URLs, responsive image sources, background images, basic image metadata and accessible frame content on the active page. This processing is limited to information already available to the browser.

How the data is used

This information is used only to list page images, apply the user's filters, manage a temporary bulk-download queue and save images the user explicitly selects.

Sharing and transmission

The extension does not transmit page scan results, image URLs, preferences or downloaded files to the developer. When the user requests an image, the browser contacts the source website or image host directly.

Chrome storage and sync

Size filters and page-control preferences are stored through Chrome sync storage. Depending on the user's Chrome sign-in and sync settings, Chrome may synchronize them between the user's devices. The complete bulk-download queue, including its items and source URLs, is kept in chrome.storage.local. These values are not sent to the extension developer. Preferences remain until changed, cleared or the extension is uninstalled. Completed queue data remains locally stored until a new queue replaces it, the user clears extension data, or the extension is uninstalled or updated in a way that removes extension data.

Data sale and prohibited uses

The extension does not sell user data, use it for advertising, or use it for creditworthiness or lending purposes.

Remote code

The extension does not load or execute remotely hosted code. All executable JavaScript is included in the installed extension package.

Limitations

PixelPull does not bypass logins, paywalls, DRM, private areas or rate limits. Users are responsible for downloading only images they own or are permitted to use.

Retention and deletion

The developer does not retain extension data because no data is sent to a developer-operated server. Completed queue items and their source URLs remain in local extension storage until a new queue replaces them, extension data is cleared, or the extension is uninstalled or updated in a way that removes extension data. Downloaded files remain in the location chosen by the user and can be deleted there.

Contact

For support or privacy questions, use the support URL on the extension's Chrome Web Store item page.
